LLM Evaluation Dashboard #

Problem Statement #

RAG systems are hard to evaluate. How do you:

  • Compare different chunking strategies (semantic vs fixed-size)?
  • Measure hallucination rates across prompt variations?
  • A/B test retrieval methods (dense vs hybrid)?
  • Track performance over time?

This project builds a comprehensive evaluation framework with visual dashboards to systematically improve RAG system quality.

Evaluation Metrics #

1. Answer Accuracy #

Compare generated answer to golden answer:

  • Exact match ‚Äî Binary correctness
  • BLEU score ‚Äî N-gram overlap
  • ROUGE score ‚Äî Recall-oriented similarity
  • Semantic similarity ‚Äî Embedding cosine distance

2. Hallucination Detection #

Check if answer contains hallucinated information:

  • Source attribution ‚Äî All facts cite retrieved documents
  • Out-of-scope detection ‚Äî Correctly says “I don’t know”
  • Factual consistency ‚Äî NLI model checks answer vs sources

3. Retrieval Quality #

Evaluate document retrieval:

  • Precision@K ‚Äî Relevant docs in top K
  • Recall@K ‚Äî Coverage of all relevant docs
  • MRR (Mean Reciprocal Rank) ‚Äî Position of first relevant doc
  • Latency ‚Äî Time to retrieve + rank documents

Tested Configurations #

Chunking Strategies #

  1. Fixed-size ‚Äî 512 tokens, 50 token overlap
  2. Semantic ‚Äî Split by paragraph/section headers
  3. Recursive ‚Äî Hierarchical chunking with parent-child

Prompt Variations #

  1. Zero-shot ‚Äî Direct question answering
  2. Few-shot ‚Äî 3 examples in prompt
  3. Chain-of-thought ‚Äî “Think step-by-step…”
  4. Constrained ‚Äî “Answer using only provided context”
  5. Structured ‚Äî JSON output format

Retrieval Methods #

  1. Dense ‚Äî Pure embedding similarity (FAISS)
  2. Hybrid ‚Äî Dense + BM25 sparse retrieval

Example Evaluation Run #

from llm_eval import RAGEvaluator

evaluator = RAGEvaluator(
    dataset="golden_qa.json",
    chunking="semantic",
    prompt_template="constrained",
    retrieval="hybrid"
)

results = evaluator.run()
# Logs to MLflow automatically

print(results.summary())
# Accuracy: 0.85, Hallucination: 0.06, Latency: 420ms

Automated Testing #

CI/CD pipeline (GitHub Actions) runs on every PR:

- Run evaluation on 20-question smoke test
- Fail if accuracy < 75%
- Fail if hallucination > 10%
- Fail if P95 latency > 1000ms

Key Learnings #

  1. Golden datasets are essential ‚Äî Can’t improve what you don’t measure
  2. Metrics must match use case ‚Äî BLEU ‚â† semantic correctness
  3. Ablation studies reveal insights ‚Äî Change one variable at a time
  4. Latency-quality tradeoffs ‚Äî Faster retrieval may hurt accuracy
  5. Visualizations drive decisions ‚Äî Dashboard convinced stakeholders

Challenges & Solutions #

Challenge Solution
Creating golden dataset Hired domain experts for annotation
Expensive LLM calls Cache LLM responses, use cheaper models for eval
Evaluation is subjective Multiple annotators, inter-rater agreement
Complex UI Streamlit with clear filters and tooltips
